# Pattern Language

> **Version:** 1.0.0
> **Status:** active
> **Last Updated:** 2026-02-08
> **Provenance:** 3P
> **Derives From:** —

Structured method for documenting recurring solutions in context, creating a network of interconnected patterns that address design problems at multiple scales. Adopted in SemOps for the pattern registry and domain pattern documentation system.

- **Standard/Origin:** Alexander 1977, GoF 1994

## Adoption Context

- **Original Capability:** `pattern-management` — methodology for pattern documentation
- **Problem Statement:** Patterns need a standardized documentation format — problem, context, forces, solution, consequences. Without methodology, patterns are just naming conventions.
- **Selection Criteria:** Christopher Alexander's pattern language (1977) + Gang of Four (1994) provide the methodology for structured pattern documentation. Foundation for `semantic-object-pattern` schema.
